Gesture education

communications, gesture education

Gesture education explains how to interact with an element using a gesture.

Types

communications, gesture education

Partial surface gesture education

Read More

A partial surface gesture affects only a portion of the screen, such as swiping a card.

Display a sample UI

To show how to use a gesture for the first time, display a sample UI that mimics the movement of the gesture being represented.

Dismissing gesture education

The only way to dismiss the sample UI should be when the user performs the gesture being demonstrated. Once a gesture is detected, the gesture indicator should pause, and then disappear.

Writing

Describe the result of the gesture briefly and clearly.

Full-screen gestures affect the entire screen, such as pinching to zoom in.

Display a banner

A banner should describe what a full-screen gesture will do.

Dismissing gesture education

Upon touching the screen, the screen content should return to its default state, with the gesture indicator removed. The banner should be displayed until the user dismisses it.

Writing

Describe the result of the gesture briefly and clearly.

Gesture education should be displayed as the user needs it, not just in the first moments of using an app. It should help users interact with an element or surface in a way they have not done so previously.

Whether the gesture affects only part of the screen, or the entire screen, similar targeting and triggering guidelines are used.

Targeting

Triggering

Show gesture education only to users who have not performed the gesture.

Gesture education occurs:

  • During the first-run experience (FRE)
  • In later sessions, if a user has not performed the gesture
  • After a major change to the UI

If a certain gesture is core to the user experience, present it at a contextually relevant moment later in the first-run experience.

If a gesture is not critical to the experience, show it in a later session at a contextually relevant moment.
